üîß Step 1: Enable SSH in WSL

1

Install and Configure SSH Server

First, you need to enable SSH server in your WSL distribution:

# Open WSL terminal and run these commands sudo apt update sudo apt install openssh-server # Start SSH service sudo service ssh start # Enable SSH to start automatically sudo systemctl enable ssh
2

Configure SSH (Optional)

For better security, you can configure SSH settings:

# Edit SSH configuration sudo nano /etc/ssh/sshd_config # Restart SSH service after changes sudo service ssh restart # Check SSH is running sudo service ssh status
3

Get WSL IP Address

Find your WSL IP address for connection:

# Get WSL IP address ip addr show eth0 # Or use hostname command hostname -I
üìù Note: WSL 2 uses a dynamic IP address that changes on restart. WSL 1 typically uses localhost (127.0.0.1).

üîß Advanced Configuration

üåê Port Forwarding (Alternative Method)

If direct IP access doesn't work, you can use port forwarding:

# In WSL - Start SSH service sudo service ssh start # In Windows PowerShell (as Administrator) netsh interface portproxy add v4tov4 listenport=2222 listenaddress=0.0.0.0 connectport=22 connectaddress=172.x.x.x

Then connect using localhost:2222 in the application.

üîë SSH Key Authentication

For better security, use SSH keys:

# In WSL - Generate SSH key pair ssh-keygen -t rsa -b 4096 # Add public key to authorized_keys cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys chmod 600 ~/.ssh/authorized_keys

Then in the web app, upload the private key file (id_rsa) for authentication.

üîÑ Auto-Start SSH Service

To automatically start SSH when WSL boots:

# Add to ~/.bashrc echo "sudo service ssh start" >> ~/.bashrc

üõ†Ô∏è Troubleshooting

üîç Connection Issues

Problem: Cannot connect to WSL
Solution:
# Test SSH connectivity from Windows ssh username@wsl-ip-address # Check if SSH is listening in WSL sudo netstat -tlnp | grep :22

üî• Firewall Issues

Problem: Windows Firewall blocking connections
Solution:
# In Windows PowerShell (as Administrator) New-NetFirewallRule -DisplayName "SCP App" -Direction Inbound -Port 5000 -Protocol TCP -Action Allow

üåê Common WSL Network Addresses

  • WSL 1: Usually 127.0.0.1 (localhost)
  • WSL 2: Dynamic IP in 172.x.x.x range (check with hostname -I)
